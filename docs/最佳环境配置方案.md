# KittenTTS 最佳环境配置方案（客观推荐）

## 项目概述

KittenTTS 是一个基于 ONNX Runtime 的文本转语音（TTS）项目，支持 CPU 和 GPU 推理。本文档从零开始，推荐最优的环境配置方案，不依赖任何现有环境。

---

## 技术需求分析

### KittenTTS 项目需求
- **ONNX Runtime**: 用于模型推理
- **ONNX Opset**: 模型使用 opset 5
- **Web 框架**: Gradio
- **语音处理**: phonemizer
- **数值计算**: numpy

### GPU 推理需求
- **CUDA**: 12.x（onnxruntime-gpu 1.17.0+ 要求）
- **cuDNN**: 9.x（onnxruntime-gpu 1.17.0+ 要求）
- **GPU**: NVIDIA GPU（支持 CUDA 12.x）

---

## 推荐方案对比

### 方案 1：Python 3.11 + onnxruntime-gpu 1.17.0 + CUDA 12.4 + cuDNN 9.x（推荐⭐⭐⭐⭐⭐⭐）

#### 配置
```
Python: 3.11.x
onnxruntime-gpu: 1.17.0
numpy: 1.26.4
CUDA: 12.4
cuDNN: 9.x
```

#### 技术优势
- ✅ **Python 3.11 稳定性**：成熟稳定，兼容性好
- ✅ **onnxruntime-gpu 1.17.0**：支持 ONNX opset 5，性能优秀
- ✅ **CUDA 12.4**：稳定版本，广泛使用
- ✅ **cuDNN 9.x**：满足 GPU 推理需求
- ✅ **GPU 加速**：20-50x 性能提升

#### 兼容性
- ✅ **ONNX opset 5**：完全支持
- ✅ **Python 3.11**：所有依赖都支持
- ✅ **CUDA 12.4**：onnxruntime-gpu 1.17.0 推荐
- ✅ **cuDNN 9.x**：满足版本要求

#### 实施步骤

##### 步骤 1：安装 Python 3.11.x

下载 Python 3.11.x：
https://www.python.org/downloads/

安装时勾选：
- ✅ Add Python to PATH
- ✅ Install for all users

##### 步骤 2：创建虚拟环境

```bash
python -m venv venv
.\venv\Scripts\activate
```

##### 步骤 3：安装依赖

```bash
pip install --upgrade pip
pip install onnxruntime-gpu==1.17.0
pip install numpy==1.26.4
pip install gradio
pip install phonemizer
pip install packaging
pip install protobuf
```

##### 步骤 4：安装 CUDA 12.4

下载 CUDA 12.4：
https://developer.nvidia.com/cuda-12-4-0-download-archive

选择：
- **Operating System**: Windows
- **Architecture**: x86_64
- **Version**: 12.4.0
- **Installer Type**: exe (local)

##### 步骤 5：安装 cuDNN 9.x

下载 cuDNN 9.x：
https://developer.nvidia.com/cudnn

需要注册 NVIDIA 开发者账号（免费）。

选择：
- **cuDNN Version**: 9.x
- **CUDA Version**: 12.x
- **Operating System**: Windows
- **Architecture**: x86_64

##### 步骤 6：解压并复制 cuDNN 文件

解压后，以管理员身份运行：

```bash
# 复制 bin 文件
copy "解压路径\cuda\bin\*.dll" "C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v12.4\bin\"

# 复制 include 文件
copy "解压路径\cuda\include\*.h" "C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v12.4\include\"

# 复制 lib 文件
copy "解压路径\cuda\lib\x64\*.lib" "C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v12.4\lib\x64\"
```

##### 步骤 7：验证安装

```bash
# 验证 CUDA
nvcc --version

# 验证 cuDNN
dir "C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v12.4\bin" | findstr cudnn

# 验证 Python 环境
python -c "import onnxruntime as ort; print(ort.get_available_providers())"
```

应该输出：
```
['TensorrtExecutionProvider', 'CUDAExecutionProvider', 'CPUExecutionProvider']
```

##### 步骤 8：测试

```bash
python web_ui.py
```

#### 性能预期
- **实时率**: 20-50x
- **配置难度**: 中等
- **稳定性**: 高

## 常见问题

### Q1: 为什么推荐 Python 3.11 而不是 3.12？
A: Python 3.11 更成熟稳定，所有依赖都完美支持。Python 3.12 虽然更新，但可能有兼容性问题。

### Q2: 为什么推荐 CUDA 12.4 而不是 12.9？
A: CUDA 12.4 是稳定版本，广泛使用，onnxruntime-gpu 推荐。CUDA 12.9 虽然更新，但可能不够稳定。

### Q3: cuDNN 必须安装吗？
A: 如果使用 GPU 推理，必须安装 cuDNN 9.x。如果使用 CPU 推理，不需要。

### Q4: 可以使用 CUDA 11.x 吗？
A: 不推荐。onnxruntime-gpu 1.17.0+ 需要 CUDA 12.x，CUDA 11.x 不兼容。

### Q5: CPU 模式性能如何？
A: CPU 模式实时率 7-8x，性能已经不错。如果对性能要求不是特别高，可以直接使用 CPU 模式。
